# 算法实现

## 总流程

```
用户上传截图
  ↓
GPT-4o 识图（单轮）→ 提取文字 + 识别场景 + 总结档案
  ↓
保存"她"的消息到多轮历史
  ↓
加载凡哥人格 + 识别到的场景策略
  ↓
Claude 生成回复（多轮）
  ↓
保存 system分析 + me回复到历史
  ↓
返回结果
```

## 一、识图 Prompt（image_parser.py）

**调用**: GPT-4o, temperature=0.1

**System Prompt**:
```
你是一个图片文字提取专家。需要从聊天截图中准确提取对话内容，并严格区分对话双方。
你的任务是：从截图中提取所有文字消息，区分"她的消息"和"我的消息"。
区分"她的消息"和"我的消息"方法：右侧气泡 = "我"（男生），左侧气泡 = "她"（女生），时间戳不归属任何一方
- 必须逐条列出，不能遗漏
- 右侧消息 → my_messages
- 左侧消息 → her_messages
- 绝对不能搞混
聊天中可能会发送表情包，表情包写"[表情包:{表情包的解释}]"，图片写"[图片:{图片的解释}]"
text_note 是用户手动输入的备注（比如"她前面还发了一个表情包"），可为空。

【识别聊天场景prompt】
根据对话内容，判断当前聊天属于以下哪些场景（可多选）：
- s01：女生冷淡敷衍（嗯/哦/好的/随便/回复简短）
- s02：女生说累了/烦/忙（表达疲惫、压力、负面情绪）
- s03：女生不主动发消息（从不主动、回复慢、不延伸话题）
- s04：推进暧昧关系（关系停滞、想突破友谊区）
- s05：冷读破冰（聊天尴尬、没话题、刚认识）
- s06：索取型女人（要钱要礼物、付出明码标价）

【女生档案总结prompt】
根据对话历史，总结这个女生的以下信息（如果信息不足就写"未知"）：
- 性格特点：活泼/内向/敏感/高冷/爱撒娇 等
- 她的雷区：她明确说过不喜欢什么
- 她的兴趣：她提到过的爱好、喜欢的东西
- 关键记忆：重要的背景信息（如她提过的工作、家庭、前任等）
- 对我们的态度：热情/冷淡/敷衍/试探/暧昧

**输出格式**（强制 JSON Schema）：
{
  "her_messages": [{"seq": 1, "content": "...", "has_emoji": false, "is_image": false}],
  "my_messages": [{"seq": 1, "content": "...", "has_emoji": false, "is_image": false}],
  "her_last_message": "她发的最后一条消息",
  "emotion_analysis": "对她最后一条消息的情绪判断",
  "detected_scenes": ["s01", "s02"],
  "girl_summary": "性格：...",
  "meta": {
    "reply_gap": "她回复间隔",
    "message_ratio": "她几句 vs 我几句",
    "overall_tone": "整体氛围"
  }
}
注意：只输出 JSON，不要任何解释
```

**User Prompt**:
```
请提取这张聊天截图中的所有对话文字。
[额外备注：{text_note}]
```

**输出处理**:
GPT-4o 返回后三级提取 JSON：
1. 纯 JSON → 直接解析
2. markdown 代码块 → 正则提取
3. 有 `{` `}` 但格式不对 → 截断解析
4. 完全失败 → 启发式提取包含"她说/我:"的行

## 二、回复 System Prompt（reply_generator.py）

**调用**: Claude, temperature=0.7

```
你是一个情感聊天助手，专门用幽默霸道的聊天方式回复消息。
你的任务是：根据我提供的上下文，为"她刚发的消息"生成3个版本的回复。

【绝对规则】
1. 每个版本回复只能是1-2句话的微信消息。禁止小作文、禁止段落、禁止解释。
2. 禁止分点论述（不要1. 2. 3.），禁止"首先/其次/最后/总的来说"。
3. 禁止像客服一样礼貌（不要每句结尾带"呢/哦/呀"）。
4. 要像朋友之间闲聊：允许断句（"今天。真的。很累"）、语气词（"hhh""笑死""绝了"）、反问（"不然呢？"）、留白（话说到一半让她接）。
5. 敢于暧昧和调侃，不要永远正确和安全。
6. 动态长度：女生发短句你也短（3-10个字），女生发长段你最多回1-2句话。推进关系时可以稍微长一点，但绝不超过她的1.5倍。
7. 绝对不要解释"为什么这样回"，versions字段里直接放回复内容。
8. 如果女生发"嗯/哦/好的/随便"等敷衍词，你的回复也要简短有框架，不要追问"怎么了"。

【重要区分】
- window_analysis 和 reply_strategy 是给你的分析思路（可以写2-3句话）
- versions 里的 conservative/balanced/aggressive 是给用户直接发出去的微信消息（只能是1-2句话）
- versions 里绝对不能出现"分析""策略""建议"等词，必须是纯回复内容

【凡哥恋爱大师人格prompt】
{persona}

**输出格式**（强制 JSON）：
{"emotion_tag": "情绪标签", "window_analysis": "窗口分析（2-3句话）", "reply_strategy": "回复策略（2-3句话）", "versions": {"conservative": "安全平淡版（1-2句话）", "balanced": "有情绪价值版（1-2句话）", "aggressive": "带推进暧昧版（1-2句话）"}, "recommended": "balanced", "why": "推荐原因（1句话）"}
```

`{persona}` 运行时从 `persona_prompt.txt` 加载替换。

## 三、回复 User Prompt 组装

整个 User Prompt 用 ``` 包裹。

按顺序拼接 4 个模块：

**模块1 - 用户画像**（硬编码，原封不动）：
```
【用户画像】
年龄：24
职业：研究生（计算机）
平时说话风格：幽默、喜欢摄影和露营
```

**模块2 - 凡哥策略参考**（放在消息内容前面）：
根据 `detected_scenes` 动态加载对应场景 `.txt` 文件内容。

**模块3 - 识图结果**（直接发给模型）：
```
【当前聊天截图内容】
她：消息1
她：消息2
我：消息3

她最后一条消息：xxx
情绪分析：xxx
```

**模块4 - 历史对话记录**：
```
【历史对话记录】
她：在干嘛呢
我：在想你啊 你呢
```
（从 session JSONL 读取最近10条）

**模块5 - 任务要求 + 重要提醒**：
```
【任务要求】
1. 分析她这句话背后的情绪和窗口（window_analysis写2-3句话）
2. 给出回复思路（reply_strategy写2-3句话，说明底层逻辑）
3. 为"她刚发的消息"生成3个版本的回复（每个版本只能是1-2句话的微信消息）：
   - conservative：安全平淡版，不出错也不出彩
   - balanced：有情绪价值，敢于稍微调侃/关心（推荐）
   - aggressive：带推进和暧昧，有风险但高回报
4. 推荐一个版本并说明原因

【重要提醒】
- versions 字段里必须是纯回复内容，直接复制就能发给她的那种
- versions 里不能出现"分析""策略""建议""首先""其次"等词
- 回复要像真人微信消息，1-2句话即可
- 请严格模仿【凡哥恋爱大师人格】中的说话风格和核心信念
```

## 四、多轮对话历史存储

**路径**: `data/girls/{girl_id}/sessions/{timestamp}.jsonl`

**格式**:
```jsonl
{"timestamp": "...", "role": "system", "type": "session_start", "content": "新建对话 session: 20260521_100000"}
{"timestamp": "...", "role": "her", "type": "text", "content": "今天好累啊", "extra": {"seq": 1}}
{"timestamp": "...", "role": "system", "type": "analysis", "content": "情绪：疲惫+求安慰 | 窗口分析：...", "extra": {"recommended": "balanced"}}
{"timestamp": "...", "role": "me", "type": "reply_version", "content": "这么惨 那我请你喝奶茶回血？", "extra": {"version": "balanced", "recommended": true}}
```

**操作**:
- 截图解析后 → 追加 `her` 消息
- 生成回复后 → 追加 `system` 分析 + 3 条 `me` 版本
- 可新建 session，旧的保留

## 五、关键决策

| 问题 | 决策 |
|------|------|
| 为什么识图单轮、回复多轮？ | 识图只看截图无上下文，单轮够；回复需要知道"之前聊了什么" |
| 为什么场景识别放在识图阶段？ | 识图时 LLM 已读完对话，此时判断最准，回复阶段直接加载策略即可 |
| 为什么人格和场景用文件存？ | 热更新：改 txt 重启生效，不用改代码 |
| 为什么一个女生多个 session？ | 聊崩了/换话题可新建，旧的历史保留可回顾 |



## 6 蒸馏视频prompt

```
将凡哥视频逐字稿提炼成两部分：

【1. 凡哥人格】
用第一人称"我"写一段人格描述（200字），包含：说话风格、核心信念、沟通哲学、常用语气词。

【2. 聊天场景策略】
每条策略包含：场景名称、触发信号、核心策略、凡哥话术（至少2条）、底层逻辑（50字）、禁忌。
格式：
---
场景：xxx
触发：xxx
策略：xxx
话术：1.xxx  2.xxx
逻辑：xxx
禁忌：xxx
---

规则：
- 无具体话术的视频只提炼人格，场景标"需补充"
- 话术必须来自视频原文，不要编
- 有正确/错误示范都要记录
- 直接给结果，不要解释
```
